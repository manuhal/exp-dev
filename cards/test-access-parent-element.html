Iframe Parent Access Notes

This page is a short reference for the iframe and parent-page question.

Plain answer: if the parent page is on a different domain, your iframe can see the parent URL string, but it cannot reach into the parent page and hide or change elements there.

So: reading the URL is allowed. Changing the parent DOM is not.

1. Same Origin

If the iframe and the parent page share the same origin, direct DOM access works. In that case, the iframe can look up the parent element and hide it.

var calendarBtn = window.parent.document.getElementById("calendar-btn");
if (calendarBtn) {
    calendarBtn.style.display = "none";
}

This only applies when both pages have the same protocol, host, and port.

2. Cross Origin

If the parent is on a different domain, direct DOM access is blocked by the browser's Same-Origin Policy. That is why a cross-origin iframe cannot hide #calendar-btn in the Experience shell.

Your case: the parent is experience.elluciancloud.com or experience-test.elluciancloud.com.

That means the iframe can still read the referrer URL, but it cannot reach into the parent page and change elements there.

If messaging is supported, the iframe can send a request and the parent page can decide whether to act on it.

window.parent.postMessage({ action: "hide", id: "calendar-btn" }, "*");

This only helps if the parent shell already listens for that message. If it does not, nothing happens.

Practical Options

  1. If #calendar-btn is inside your iframe, hide it directly with JavaScript.
  2. If #calendar-btn is in the Experience shell, you cannot hide it from the iframe.
  3. Ask Ellucian whether the platform supports a card API, shell hook, or supported message listener.
  4. Use parent-side CSS or configuration only if you control the shell itself.

Bottom Line

Cross-origin iframe access is intentionally restricted by all modern browsers. The browser lets you observe some context, like the referrer URL, but it will not let your iframe directly manipulate the parent DOM.

That restriction is the security boundary that keeps a card from tampering with the host page.
